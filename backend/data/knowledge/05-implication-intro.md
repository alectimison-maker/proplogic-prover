# 条件证明（→-intro）

## 概述

**→-intro**（蕴含引入 / 条件证明 / Conditional Proof）是自然演绎中引入蕴含的规则。
它是第一个需要使用**子证明**（sub-proof）的规则，理解子证明的概念对后续学习至关重要。

---

## 一、蕴含引入（→-intro）

### 形式定义

        | A          假设
        | ...
        | B          （在假设 A 下推导出 B）
        ─────────
        A → B        →-intro

### 详细说明

要证明 A → B（"如果 A 则 B"），按如下步骤操作：

1. **假设** A 成立（开启一个子证明）
2. 在这个假设下，通过一系列推理**推导出** B
3. **关闭**子证明，得到结论 A → B

直觉理解：要证明"如果 A 则 B"，我们临时假设 A 成立，然后展示在这个假设下 B 一定成立。由于 B 是在假设 A 下得出的，所以 A → B 成立。

### 使用格式（Fitch 风格）

    n.   | A              假设（assumption）
    ...  | ...            （在假设下推导）
    m.   | B              （得到目标后件）
    k.   A → B            →-intro n-m

- 竖线 | 表示子证明的范围
- →-intro 引用的行号范围是 n-m，即从假设行到结论行
- 关闭子证明后，子证明内部的行**不能**再被外层直接引用

### 关于子证明的重要规则

1. **子证明内可以引用外层的任何行**（包括前提和外层子证明中的行）
2. **子证明外不能引用子证明内部的行**（假设和假设下的推导在关闭后失效）
3. 子证明可以**嵌套**：在子证明内部再开一个子证明
4. 假设行只在当前子证明内有效

---

## 二、子证明的结构

### 单层子证明

    1. X               前提
    2. | A              假设
    3. | ...            推导过程
    4. | B              得到 B
    5. A → B            →-intro 2-4

### 嵌套子证明（两层）

    1. X               前提
    2. | A              假设（外层子证明）
    3. | | B            假设（内层子证明）
    4. | | ...          推导过程
    5. | | C            得到 C
    6. | B → C          →-intro 3-5（关闭内层）
    7. A → (B → C)      →-intro 2-6（关闭外层）

嵌套子证明用于证明形如 A → (B → C) 的结论：
- 外层假设 A
- 内层假设 B
- 推导出 C
- 关闭内层得 B → C
- 关闭外层得 A → (B → C)

## 三、完整示例

### 例题 1（P56）：¬Q → ¬P ⊢ P → ¬¬Q

**目标**：从前提 ¬Q → ¬P 推导出 P → ¬¬Q。

**分析**：
- 目标 P → ¬¬Q 是蕴含式，所以用 →-intro
- 假设前件 P，需要推导出后件 ¬¬Q
- 有前提 ¬Q → ¬P，且假设了 P
- 对 P 使用 ¬¬-intro 得到 ¬¬P
- 对 ¬Q → ¬P 和 ¬¬P 使用 MT：A = ¬Q, B = ¬P, ¬B = ¬¬P，得 ¬A = ¬¬Q

**证明**：

    1. ¬Q → ¬P         前提
    2. | P              假设
    3. | ¬¬P           ¬¬-intro 2
    4. | ¬¬Q           MT 1, 3
    5. P → ¬¬Q         →-intro 2-4

**步骤解析**：

- **第 1 行**：写出前提
- **第 2 行**：开启子证明，假设目标蕴含的前件 P
- **第 3 行**：对假设的 P 使用 ¬¬-intro，得到 ¬¬P。这一步是为了配合 MT 使用
- **第 4 行**：使用 MT。蕴含式 ¬Q → ¬P 的后件是 ¬P，其否定是 ¬¬P（第 3 行）。所以 MT 得到前件 ¬Q 的否定，即 ¬¬Q
- **第 5 行**：关闭子证明。我们在假设 P 的条件下得到了 ¬¬Q，因此 →-intro 给出 P → ¬¬Q

### 例题 2（P58）：(P ∧ Q) → R ⊢ P → (Q → R)

**目标**：这是**柯里化**（currying）的证明——将双参数蕴含转化为嵌套蕴含。

**分析**：
- 目标 P → (Q → R) 是嵌套蕴含，需要两层子证明
- 外层假设 P，需推导 Q → R
- 内层假设 Q，需推导 R
- 在两层假设下，有 P 和 Q，可以用 ∧-intro 得到 P ∧ Q
- 有 (P ∧ Q) → R 和 P ∧ Q，用 →-elim 得到 R

**证明**：

    1. (P ∧ Q) → R      前提
    2. | P               假设（外层子证明）
    3. | | Q             假设（内层子证明）
    4. | | P ∧ Q         ∧-intro 2, 3
    5. | | R             →-elim 1, 4
    6. | Q → R           →-intro 3-5
    7. P → (Q → R)       →-intro 2-6

**步骤解析**：

- **第 2 行**：开启外层子证明，假设 P
- **第 3 行**：开启内层子证明，假设 Q
- **第 4 行**：在两层假设下，P（第 2 行）和 Q（第 3 行）都可用，用 ∧-intro 合并得 P ∧ Q。注意第 2 行虽然在外层子证明中，但内层可以引用
- **第 5 行**：用 →-elim 将前提 (P ∧ Q) → R（第 1 行）与 P ∧ Q（第 4 行）结合，得到 R。注意前提第 1 行在子证明外，但子证明内可以引用
- **第 6 行**：关闭内层子证明。假设 Q 推导出 R，所以 Q → R
- **第 7 行**：关闭外层子证明。假设 P 推导出 Q → R，所以 P → (Q → R)

---

### 例题 3：⊢ P → P（无前提证明）

**目标**：证明 P → P（任何命题蕴含自身），不需要任何前提。

**证明**：

    1. | P               假设
    2. P → P             →-intro 1-1

这是最简单的 →-intro 使用：假设 P，P 本身已经在假设中，所以直接关闭子证明。

## 四、解题策略

### 4.1 目标导向（Goal-directed）

当目标是 A → B 时：
1. 开启子证明，假设 A
2. 想办法推导出 B
3. 关闭子证明，得到 A → B

### 4.2 嵌套蕴含的处理

当目标是 A → (B → C) 时：
1. 假设 A（外层）
2. 假设 B（内层）
3. 推导 C
4. 关闭内层得 B → C
5. 关闭外层得 A → (B → C)

### 4.3 与 →-elim 的配合

→-intro 和 →-elim 经常配合使用：
- 在子证明内部，用 →-elim 消除已有的蕴含
- 在子证明外部，用 →-intro 引入新的蕴含

### 4.4 确定假设什么

- 假设的内容由**目标蕴含的前件**决定
- 如果目标是 P → Q，就假设 P
- 如果目标是 ¬P → ¬Q，就假设 ¬P

## 五、常见错误

### 错误 1：在子证明外引用子证明内的行

    1. A → B            前提
    2. | A               假设
    3. | B               →-elim 1, 2
    4. A → B             →-intro 2-3
    5. C → B             →-intro ?, 3 ✗  ← 第 3 行在已关闭的子证明中，不可引用

子证明关闭后，其内部的行号不再可用。

### 错误 2：忘记开启子证明

    1. A → B            前提
    2. A                 ← 这不是假设！这是在声明 A 为真
    3. B                 →-elim 1, 2
    4. A → B             →-intro 2-3 ✗

假设必须明确标记为 assumption，并用竖线标示子证明范围。

### 错误 3：子证明嵌套层次错误

    目标：P → (Q → R)

    1. | P               假设
    2. | Q               ← 应该是新的子证明层级 | | Q
    ...

对于嵌套蕴含，每一层蕴含需要自己的子证明层级。

## 六、练习

本章内容对应题库中的以下习题：

- **题库第 16-21 题**：条件证明练习
- 这些习题需要使用 →-intro 以及之前学过的所有规则
- 部分习题需要嵌套子证明

**练习策略**：
1. 看到目标是 A → B 的形式，立刻想到 →-intro
2. 假设 A，目标变为推导 B
3. 在子证明内，综合使用前提和假设，运用已学的所有规则
4. 如果推导中需要用到 MT，注意可能需要先做 ¬¬-intro 转换
5. 嵌套蕴含目标需要多层子证明，从外到内逐层假设
