# 蕴含消除（→-elim）及相关规则

## 概述

蕴含消除是命题逻辑中最常用的推理规则之一。本章介绍四条与蕴含消除相关的规则：
- **→-elim**（肯定前件式 / Modus Ponens）
- **MT**（否定后件式 / Modus Tollens）
- **HS**（假言三段论 / Hypothetical Syllogism）
- **DS**（析取三段论 / Disjunctive Syllogism）

---

## 一、蕴含消除 / 肯定前件式（→-elim / Modus Ponens）

### 形式定义

        A → B    A
        ────────────  →-elim
             B

### 详细说明

如果你有一个蕴含 A → B（"如果 A 则 B"），并且你已经证明了前件 A，就可以推出后件 B。

这是最基本、最重要的推理规则。它的日常逻辑非常自然：
- "如果下雨，地就湿"（P → Q）
- "现在在下雨"（P）
- 因此"地是湿的"（Q）

### 使用格式

    n.  A → B      （已有）
    m.  A          （已有）
    k.  B          →-elim n, m

注意：→-elim 需要引用两个行号——蕴含式和前件。两者的顺序在不同系统中可能不同，本系统中先写蕴含式的行号，再写前件的行号。

### 重要提醒

- 必须**精确匹配**：前件 A 必须与 → 左边的内容完全一致
- 如果蕴含式是 (P ∧ Q) → R，那么前件必须是 P ∧ Q，不能只是 P 或只是 Q

---

## 二、否定后件式（MT / Modus Tollens）

### 形式定义

        A → B    ¬B
        ────────────  MT
            ¬A

### 详细说明

如果你有 A → B，并且后件 B 的否定 ¬B 成立，那么前件 A 的否定 ¬A 也成立。

直觉理解：
- "如果下雨，地就湿"（P → Q）
- "地没有湿"（¬Q）
- 因此"没有下雨"（¬P）

MT 本质上是蕴含的**逆否**推理：A → B 等价于 ¬B → ¬A。

### 使用格式

    n.  A → B      （已有）
    m.  ¬B         （已有）
    k.  ¬A         MT n, m

### 特殊情况

当蕴含式的前件或后件本身带有否定时，MT 的结果会出现双重否定：

    已有：¬Q → ¬P 和 ¬¬P
    MT 得到：¬¬Q

这里 A = ¬Q, B = ¬P，¬B = ¬¬P，¬A = ¬¬Q。
如果需要 Q 而不是 ¬¬Q，后续可以用 ¬¬-elim 消除。

---

## 三、假言三段论（HS / Hypothetical Syllogism）

### 形式定义

        A → B    B → C
        ────────────────  HS
             A → C

### 详细说明

如果 A 蕴含 B，B 蕴含 C，那么 A 蕴含 C。这是蕴含的**传递性**。

直觉理解：
- "如果下雨，地就湿"（P → Q）
- "如果地湿，就会滑"（Q → R）
- 因此"如果下雨，就会滑"（P → R）

### 使用格式

    n.  A → B      （已有）
    m.  B → C      （已有）
    k.  A → C      HS n, m

注意：第一个蕴含的后件必须与第二个蕴含的前件完全一致（都是 B）。

---

## 四、析取三段论（DS / Disjunctive Syllogism）

### 形式定义

        A ∨ B    ¬A            A ∨ B    ¬B
        ────────────  DS       ────────────  DS
             B                      A

### 详细说明

如果你有 A ∨ B（A 或 B 至少一个成立），并且你知道其中一个不成立，那么另一个必定成立。

直觉理解：
- "今天是周六或周日"（P ∨ Q）
- "今天不是周六"（¬P）
- 因此"今天是周日"（Q）

### 使用格式

    n.  A ∨ B      （已有）
    m.  ¬A         （已有）
    k.  B          DS n, m

或者：

    n.  A ∨ B      （已有）
    m.  ¬B         （已有）
    k.  A          DS n, m

---

## 五、完整示例

### 例题 1（P51）：P, P → Q, P → (Q → R) ⊢ R

**目标**：从三个前提推导出 R。

**分析**：
- 有 P 和 P → Q，可以用 →-elim 得到 Q
- 有 P 和 P → (Q → R)，可以用 →-elim 得到 Q → R
- 有 Q 和 Q → R，再用 →-elim 得到 R

**证明**：

    1. P               前提
    2. P → Q           前提
    3. P → (Q → R)     前提
    4. Q               →-elim 2, 1
    5. Q → R           →-elim 3, 1
    6. R               →-elim 5, 4

**步骤解析**：

- **第 4 行**：用 →-elim，蕴含式是第 2 行 P → Q，前件是第 1 行 P，得到后件 Q
- **第 5 行**：用 →-elim，蕴含式是第 3 行 P → (Q → R)，前件是第 1 行 P，得到后件 Q → R
- **第 6 行**：用 →-elim，蕴含式是第 5 行 Q → R，前件是第 4 行 Q，得到后件 R

注意第 1 行的 P 被使用了两次——同一个已证明的命题可以被多次引用。

---

### 例题 2（P52）：P → (Q → R), P, ¬R ⊢ ¬Q

**目标**：从三个前提推导出 ¬Q。

**分析**：
- 有 P 和 P → (Q → R)，用 →-elim 得到 Q → R
- 有 Q → R 和 ¬R，用 MT 得到 ¬Q

**证明**：

    1. P → (Q → R)     前提
    2. P               前提
    3. ¬R              前提
    4. Q → R           →-elim 1, 2
    5. ¬Q              MT 4, 3

**步骤解析**：

- **第 4 行**：→-elim 将 P → (Q → R) 和 P 结合，得到 Q → R
- **第 5 行**：MT 将 Q → R 和 ¬R 结合。根据 MT 的规则：有 A → B 和 ¬B，得 ¬A。这里 A = Q, B = R，¬B = ¬R，所以得到 ¬A = ¬Q

这个例子展示了 →-elim 和 MT 的配合使用。

---

### 例题 3：P → Q, Q → R, R → S ⊢ P → S

**目标**：用假言三段论链接三个蕴含。

**证明**：

    1. P → Q           前提
    2. Q → R           前提
    3. R → S           前提
    4. P → R           HS 1, 2
    5. P → S           HS 4, 3

---

### 例题 4：P ∨ Q, ¬P, Q → R ⊢ R

**目标**：综合使用 DS 和 →-elim。

**证明**：

    1. P ∨ Q           前提
    2. ¬P              前提
    3. Q → R           前提
    4. Q               DS 1, 2
    5. R               →-elim 3, 4

---

## 六、规则对比

| 规则     | 前提            | 结论   | 直觉说明             |
|----------|-----------------|--------|----------------------|
| →-elim   | A → B, A        | B      | 已知"若 A 则 B"，且 A |
| MT       | A → B, ¬B       | ¬A     | 后件假则前件假       |
| HS       | A → B, B → C    | A → C  | 蕴含的传递链接       |
| DS       | A ∨ B, ¬A       | B      | 排除一个选项         |

---

## 七、常见错误

### 错误 1：肯定后件（Affirming the Consequent）

    P → Q, Q ⊢ P     ✗ 谬误！

这不是有效推理！"如果下雨地就湿"和"地湿了"不能推出"在下雨"（可能是洒水车）。
有效的是 →-elim（肯定前件）和 MT（否定后件）。

### 错误 2：否定前件（Denying the Antecedent）

    P → Q, ¬P ⊢ ¬Q   ✗ 谬误！

"如果下雨地就湿"和"没下雨"不能推出"地没湿"。

### 错误 3：MT 结果忘记加否定

    P → Q, ¬Q
    P              MT ✗   ← 应该是 ¬P

MT 的结论是前件的**否定**，不是前件本身。

### 错误 4：→-elim 前件不匹配

    (P ∧ Q) → R, P
    R              →-elim ✗   ← 前件是 P ∧ Q，不是 P

蕴含式 → 左边是什么，前件就必须是什么，必须精确匹配。

## 八、练习

本章内容对应题库中的以下习题：

- **题库第 9-15 题**：蕴含消除及相关规则练习
- 这些习题涉及 →-elim、MT、HS、DS 以及之前学过的所有规则
- 重点练习识别何时用 →-elim、何时用 MT

**练习策略**：
1. 有 A → B 和 A → 用 →-elim 得 B
2. 有 A → B 和 ¬B → 用 MT 得 ¬A
3. 有两个首尾相接的蕴含 → 用 HS 链接
4. 有 A ∨ B 和某一选项的否定 → 用 DS 得另一选项
5. 多条蕴含链时，考虑先 →-elim 展开再继续推理
